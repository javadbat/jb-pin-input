<Title>Styling</Title>

`JBPinInput` can be styled at two levels. Start with CSS custom properties for normal design-system theming. Use exported `::part(...)` selectors when a design needs direct access to the row, cells, inner inputs, bottom lines, or message.

The gallery below is intentionally split in that order: six simple demos using only CSS variables first, then six demos that use parts. The grid is capped at two demos per row so the examples stay readable on smaller screens.

<JBPinInputStyleTest />

## CSS Variables

Use variables when you want the component to keep its internal structure and only change the design tokens.

```css
.pin-token-minimal {
  --jb-pin-input-inputs-wrapper-width: 13.75rem;
  --jb-pin-input-inputs-gap: 0.5rem;
  --jb-pin-input-bottom-line-height: 2px;
  --jb-pin-input-bottom-line-color-active: #2563eb;
  --jb-pin-input-pin-font-size: 1.375rem;
}
```

## Parts

Use parts when you need to style a specific exposed element, such as every cell wrapper or one indexed line.

```css
.pin-part-ticket jb-pin-input::part(input-line-2),
.pin-part-ticket jb-pin-input::part(input-line-3) {
  background: #fb923c;
}
```

## Usage

```tsx
<section className="pin-token-boxed">
  <JBPinInput
    charLength={4}
    value="2846"
    message="Bordered cells without part selectors."
  />
</section>
```
